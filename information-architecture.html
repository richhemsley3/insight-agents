Flow 1: Focus — Agent Surfaces a Priority Issue
User encounters a ranked insight card on their dashboard, expands reasoning to understand blast radius, and takes action or dismisses. Agent learns from the response.
ⓘ Design decision: Progressive disclosure (summary, then reasoning, then full data) keeps the default view scannable while supporting deep investigation when needed.

! Edge Cases

  • Agent suggestion is wrong — user rejects, agent records feedback and adjusts future scoring
  • Low confidence — card shows "Needs human review" badge instead of a suggested action
  • Conflicting signals — card explains uncertainty with dual-hypothesis view and asks user to choose
  • First-time user — onboarding overlay explains card anatomy before showing real data

! Error States

  • Stale insight — card shows last-refreshed timestamp with "Refresh" action
  • Action fails — inline error with rollback option and retry button
  • No insights available — empty state with tips and "Check back later" message
  • Agent offline — banner with status and estimated recovery time
Flow 2: Find — Natural Language Data Search
User asks a question in natural language. The Find agent searches the catalog, policies, and lineage, returning ranked results with context and related query suggestions.
ⓘ Design decision: Trust through transparency — every result shows confidence level, source attribution, and reasoning chain so users can verify before acting.

! Edge Cases

  • Ambiguous query — agent asks a clarifying question before searching
  • Bulk results — paginated list with "Show more" and faceted filters
  • Sensitive data matches — results flagged with access control indicators
  • Cross-domain query — agent explains when results span multiple data domains

! Error States

  • No results — empty state with alternative query suggestions and tips
  • Partial results — shows available matches with note about unavailable sources
  • Timeout — progress indicator with "Still searching..." and cancel option
  • Permission denied — explains which results are restricted and who to contact
Flow 3: Automate — Agent Suggests and Executes Actions
Agent detects an opportunity (e.g., unclassified PII columns), surfaces a suggestion card with preview, and executes after user approval. Shows before/after results.
ⓘ Design decision: Autonomy dial lets users control proactiveness per agent type — from fully manual review to auto-apply above confidence thresholds.

! Edge Cases

  • Bulk actions — agent groups similar suggestions for batch approval with "Select all" option
  • Partial success — some items in a batch fail; shows per-item status with retry for failures
  • Dependency chain — action requires prerequisite; agent explains and offers to run both sequentially
  • Auto-apply conflict — two auto-rules produce conflicting results; agent pauses and asks for human input

! Error States

  • Execution failure — rollback option with detailed error log and retry button
  • Rate limiting — queues pending actions with progress bar and estimated completion
  • Permission error — explains what permission is needed and links to admin contact
  • Stale suggestion — marks as outdated with "Re-analyze" action
Flow 4: Configure — User Adjusts Agent Behavior
User opens Agent Settings to control autonomy levels, confidence thresholds, notification preferences, and per-agent-type behavior. Changes take effect immediately.
ⓘ Design decision: Per-agent autonomy dial (Manual / Suggest / Auto-apply) with confidence thresholds gives users granular control and builds trust incrementally.

! Edge Cases

  • Team-wide vs personal — settings may be locked by an admin; shows which are overridable
  • Reset to defaults — confirmation dialog with preview of what will change
  • Conflicting thresholds — warns if a user sets contradictory rules
  • First-time setup — guided wizard with recommended defaults based on role

! Error States

  • Save failure — retains unsaved changes in form with retry option
  • Permission denied — explains admin-locked settings with "Request change" link
  • Invalid threshold — inline validation with suggestion for valid range
  • Sync delay — shows "Applying changes..." spinner with timeout fallback
